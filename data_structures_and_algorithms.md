# 线性表(List)
- 从"1"开始的。

## 基本
- 具有线一样的性质。
    - 由零个或者多个数据元素组成的有限序列。

    1. 抽象数据类型( abstract data type , ADT )


    例如：

    栈（Stack）：LIFO 结构，操作：push()、pop()、top()、isEmpty()。

    队列（Queue）：FIFO 结构，操作：enqueue()、dequeue()、isEmpty()。

    2. Operation
        Initlist(*L): 初始化操作，建立一个空的线性表L。
        ListEmpty(L): 判断线性表是否为空表，若线性表为空表，返回true,否则返回false.
        ClearList(*L): 将线性表清空。
        GetElem(L,i,*e): 将线性表中的第i个位置元素值返回给e。
        LocateElem(L,e): 在线性表中查找与给定值e相等的元素，
                        如果查找成功，返回该元素在表中序号，表示成功；
                        否则，返回0表示失败。
        ListInsert(*L,i,e): 在线性表中第i个位置插入新元素e.
        ListDelete(*L, i, *e): 删除线性表L中第i个位置元素，并用e返回其值。
        ListLength(L): 返回线性表L的元素个数。

## 线性表的两种存储结构
- hhh
    1. 线性表的顺序存储结构
        #define MAXSIZE 20
        typedef int ElemType;
        typedef struct{
            ElemType data[MAXSIZE];
            int length;
        }SqList;

        1.  - 顺序存储结构封装需要三个属性：
            - 存储空间的起始位置，数组data，它的存储位置就是线性表存储空间的存储位置。
            - 线性表的最大存储容量：数组的长度MaxSize.
            - 线性表的当前长度：length
            - 数组的长度与线性表当前的长度需要区分一下:
              数组的长度是存放线性表的存储空间的总长度，一般初始化后是不变的。
              而线性表的当前长度是线性表中元素的个数，是会变化的。
            
        2. 假设ElemType占用的是c个存储单元(字节)，那么第(i+1)和第i个数据元素的存储位置的关系是：
            Loc(a(i+1)) = Loc(ai) + c;


        3. 插入和删除
            - 线性表的顺序存储结构，在存、读数据时，不管时那个位置，时间复杂度都是O(1).
            而在插入或者删除时，时间复杂度都是O(n).

    2. 线性表的链式存储结构。

      - 特点：用一组任意的存储单元存储线性表的数据元素，这组存储单元可以存在内存中未被占用的任意位置。

      - 对比：比起顺序存储结构，每个数据单元只需要存储一个位置就可以了。
              现在链式存储结构中，除了要存储数据元素信息外，
              还要存储它的后继元素的存储地址(指针)。
      - 数据域：存储数据元素信息的域称为数据域，
        指针域：存储直接后继位置的域叫指针域。
        这两部分信息组成数据元素称为存储映像，称为结点。
      - n个结点链接成一个链表，即为线性表的链式存储结构。
      因为此链表的每个结点中只包含一个指针域，所以叫做单链表。

    3. 单链表
      1. 链表中的第一个结点的存储位置叫做头指针，最后一个结点指针为空(NULL)。
      2. 头结点和头指针的异同。
         - 头指针：
           1. 头指针是链表中指向第一个结点的指针，若链表有头结点，则是指向头结点的指针。
           2. 头指针具有标识作用，所以常用有头指针冠以链表的名字(指针 变量的名字)
           3. 无论链表是否为空，头指针均不为空。
           4. 头指针是链表的必要元素。
         - 头结点：
           1. 头结点放在第一个元素的结点之前，其数据域一般无意义(但也可以用来存放链表的长度)。
           2. 头结点不是必要的。

        - 头指针指向头结点，头结点的数据域是空的。

      3. 假设p是指向线性表第i个元素的指针，
        则该结点ai的数据域可以用p->data表示，p->data的值是一个数据元素。
        p-next表示结点ai的指针域(是指向一个结点的，而不是数据)，p->next的值是一个指针。
        (p->next也就是指向ai+1的指针。)
        (p->next->data = ai+1)。

      4. 二级指针使用
      
      - 错误示范：

          Status LinkListInit(LinkList L){

          L = (LinkList)malloc(sizeof(Node));//头结点定义。

              return ERROR;
          }

      }


      int main(){
          LinkList L;
          LinkListInit(L);

              printf("OK");
          }else{
              printf("Failure");
          }

      在三次输出中，三次的地址都是不一样的，并且不在函数里面的那个L输出指针是0；
      这是因为函数里面的形参在分配完内存就销毁了，所以看似成功了，但是失败了。


      - 正确示范：
      Status LinkListInit(LinkList *L){

          *L = (LinkList)malloc(sizeof(Node));//头结点定义。

          if(*L == NULL){
              return ERROR;
          }

          printf("%p\n",*L);

          (*L)->Next = NULL;

          return OK;
      }


      int main(){
          LinkList L;
          LinkListInit(&L);

          printf("%p\n",L);

          return 0;
      }

      这才是正确的，函数中的和主函数中参数是不一样的，函数中的形参是&L，主函数中就是L，但是在函数中都用的是*L，就是
      主函数中的L，所以相当于在修改主函数的地址，被销毁的是形参本身，不是形参指向的内容。

        形参p（指针变量）在栈上，函数结束被销毁

        但通过p修改的内容（main中的x）不会被销毁

        同样在链表初始化中：

        形参L（二级指针）在栈上，函数结束被销毁

        但通过*L修改的内容（main中的L）是永久性的

# 栈

- 栈是一个后进先出的线性表，要求只在表尾进行删除和插入操作。
- 栈就是个特殊的线性表：
   - 后进先出
   - 只能在表尾进行操作。
   - 对于栈来说，表尾称为栈的栈顶(top),相应的表头称为栈底(bottom)。
   (手枪弹夹类似，先进后出。)

- 由于栈的本质就是个线性表，所以也分为顺序存储结构和链式存储结构。

1. 栈的顺序存储结构：
    开始栈中不包含任何数据，叫做空栈，此时栈顶就是栈底。
    数据从栈顶(表尾)进入，栈顶栈底分离，整个栈的当前容量变大。
    数据出栈时从栈顶弹出，栈顶下移，整个栈的当前容量变小了。

    - 其实就现在来说，也就是和线性表一样的，表头不会动，也就是栈底不动，只有栈顶(表尾)动。

    typedef struct{
        ElemType *base;  //指向栈底的指针变量

    }

- 暴力匹配算法/一种大量回溯的算法

主串为S(长度为N)，模式串为T(长度为M)。
一个一个进行匹配，最极端情况要匹配 (N-M+1)*M 次所以

2. 字符串的公共前后缀
    1. 前缀 
    前缀是不包含最后一个字符的所有以第一个字符开头的连续子串。
    例如 'ABABA',就是： A,AB,ABA,ABAB.
    2. 后缀
    后缀是指不包含第一个字符的所有以最后一个字符结尾的连续子串。
    例如 'ABABA',就是： BABA,ABA,BA,A.
    3. 公共前后缀
    例如上面的那个，公共的有: aA,ABA.

在KMP算法中使用的前缀为真前缀，即不包括原字符串abcd的前缀。（真前缀：a, ab, abc）

- 现在简单举例子：
 例如主串S:  abababababca    (索引为i)
   模式串T:  ababac      (索引为j)
   
    第一次失配：ababa *b* ababca
               ababa *c*
    在i=5，j=5时候失配,所以按照next数组，T串向右移动三位

    移动完成后继续匹配，这时候的匹配规则才是最牛的一点：

    是从
    主串S失配的位置
    和
    模式串T向右移动了几位后的那一位
    开始进行匹配，
    所以即使后面依旧有对齐动作，但是主要是用来提示向右移动了几位，
    而并不是要按照对齐之后的位置进行匹配。

    在这里，也就是要从i=5,和,j=3开始匹配。

    由于是移动了三位，所以在T串中的前五位中，前三位和后三位是一样的，
    由于在第一次失配时，
    j=2~4 和 i=2~4 是匹配的，
    又从前缀表中得知，j=0~2 和 j=2~4 是一样的。
    所以就可以知道 i=2~4 和 j=0~2 是一样的，

    所以直接从 i=5 和 j=3 开始匹配了。

    继续匹配：ababa bababca
                aba bac
    
    发现失配：ababa ba *b* abca
                aba ba *c*
    (标注**为失配处)

    i=7 和 j=5失配了，继续向右移动三个。

    这次从i=7 和 j=3开始匹配了。

    继续匹配：abababa ba *b* ca
                   aba ba *c*

    再次找到失配处，继续进行匹配

    在 KMP 算法中，匹配过程会在以下两种情况下结束：

    匹配何时结束呢：

    匹配成功：

    当模式串指针 j 达到模式串长度 m（即 j == m）时，说明模式串的所有字符都已成功匹配主串中的连续一段。此时算法可以返回匹配的起始位置（i - j），并通常终止匹配过程（若只需找第一个匹配）。

    匹配失败（主串已扫描完毕）：
    
    当主串指针 i 达到主串长度 n（即 i == n），但模式串指针 j 尚未达到 m（j < m），说明主串中不存在完整的模式串匹配。此时循环结束，匹配失败。

### 树

- 树是n个结点的有限集。n=0时为空树，在任意一颗非空树中：
    - 有且仅有一个特定的称为根的结点；
    - 当n>1时，其余结点可分为m个互不相交的有限集T1,T2,T3,T4,......,Tm，其中每一个集合本身又是一棵树，并且称为根的子树(SubTree)。
    - 树拥有几个子树，度就是几。
        - 度为零的结点，称为叶节点或终端节点
        - 度不为零，老子不管了。
- 如果将树中结点的各个子树的顺序固定了，就是有序树。

### 二叉树
- 其实就是不能有多于两个孩子。(没有子树或者有一棵树都是可以的)

- 二叉树时n个结点的有限集合，该集合或者为空集(空二叉树)，或者由一个根结点和两棵互不相交的、分别称为根结点的左子树和右子树的二叉树组成。

- 左右子树是有顺序的，次序不能颠倒。
- 即使树中某结点只有一颗子树，也要区分它是左子树还是右子树。

#### 特殊二叉树

1. 满二叉树
- 就是都满喽，除了叶子都有两个孩子。而且所有的叶子都要在同一层上面。

2. 完全二叉树

若给满二叉树进行按照层序编号，有另一个二叉树，也进行编号，
在对应的位置上，和满二叉树的编号都一样，那么就称为完全二叉树。

完全二叉树的特点：
- 叶子结点只能出现在最下面的两层。
- 最下层的叶子一定集中在左部连续位置。
- 倒数第二层，若有叶子结点，一定都在右部连续位置。
- 如果某个结点的度为1，则该结点只有左孩子。

- 完全二叉树的结点树的范围在 与之同深度的满二叉树 和 少一层深度的二叉树 之间。

满二叉树一定是完全二叉树，但是反过来不一定对。
