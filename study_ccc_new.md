     char类型也可以储存数字并输出，%d则输出数字，%c则输出ASC||中对应的或者是乱码。
     当把一个超出范围的值赋给 char 时，会发生溢出截断。计算机只会取低 8 位（因为 char 只有 8 位），然后把那 8 位二进制数按照有符号数的规则（补码） 来解释。

     170 的二进制（int）：00000000 00000000 00000000 10101010
     截断给 char（取低8位）：10101010
     现在，height 里面存的是 10101010。
     由于 char 是有符号的，最高位是 1，所以计算机认为这是一个负数，并且是按照补码存储的。

    （正数的补码 = 它本身（与原码相同））

    - 若为signed int ,则是有符号整型，有正有负。
      则11111111 ， 第一位表示是负值，计算机中使用补码规则，所以即为00000000+1=00000001，即为-1。
    （负数的补码 = 在原码的基础上，符号位不变，其余位取反，最后加1。）

# 指针

1. 基本：

  每一个地址占用一个字节。
  每个存储单元就是一个地址，占用一个字节。
  只有编译器知道变量名对应的相应地址，在访问时可以找到。

  指针变量存放地址，普通变量存放数据。
  指针变量的类型就是存放的地址指向的数据类型。

  举个栗子：

  现在定义指针变量pa，pa中存放的是普通变量a的地址，然后再通过这个地址找到变量a对应的数据。

  其中，pa中存放的地址叫做指针，pa这个变量叫做指针变量。

  定义指针变量 : 类型名 *指针变量名
      char *pa;//定义一个指向字符型的指针变量。
      int *pb;//定义一个指向整型的指针变量。

      这个“类型名”表示的是“指针变量中存放的地址指向的内存单元的数据类型”。

  如果需要获取某个变量的地址，可以使用取地址运算符(&)。

  指针变量中存放的就是地址，所以也就是把地址赋值给了指针变量。

  char *pa = &a;
  int *pb = &f;

  注：这里在定义指针变量时候用到的*，**并不是取值运算符**，而只是定义用到的。

  如果需要访问指针变量指向的数据，可以使用取值运算符(*)。

  取值运算符是  取指针变量指向的那个变量的值。

  printf("%c, %d\n", *pa, *pb);

  - 直接通过变量来访问变量的值   叫直接访问。
  - 通过指针变量所指向的地址所指向的地址来访问   间接访问。



  - 避免访问未初始化的指针。
      例如：
          int *a;
          *a = 123;

          很危险的！

# 动态内存

1. malloc
  
    - 函数原型：void *malloc(size_t size);

    - 也可以分配多个的：(int *)malloc(sizeof(int) * num);
    > 不进行初始化操作，里面的数据是随机的。

    malloc函数向系统申请分配size个字节的内存空间，并返回一个指向这块空间的指针。
    malloc申请的空间位于内存的堆上。(栈和堆不一样)
    > 堆上的内存如果不去主动释放，就会永远存在，直到程序被关闭。
     在不再使用的时候，一定要主动释放内存，否则会内存泄漏。

    若调用成功，返回void类型指针，可以被转换成任何类型的数据。
    若函数调用失败，返回值是NULL。
    若size大小为0，则也可能返回NULL，但是这并不表示调用失败。

2. free

    - 函数原型：void *free(void *ptr);

    free函数释放ptr参数指向的内存空间。该内存空间必须是由malloc,calloc,realloc函数申请的(必须是堆内存)。
    否则，该函数将导致未定义行为。
    如果ptr参数是NULL，则不会执行任何操作。
    (该函数并不会修改ptr参数的值，所以调用后它仍然指向原来的地方，只是变为了非法空间。)

3. 内存泄漏
  1. 隐式内存泄漏(用完内存后，没有使用free函数释放)。
  2. 丢失内存块地址。

  举个栗子：

    int main(){
      int *ptr;
      int num = 123;

      ptr = (int *)malloc(sizeof(int));
      if(ptr == NULL){
          return ERROR;
      }
      printf("Please enter a number:");
      scanf("%d",ptr);
      printf("the number:%d",*ptr);

      ptr = &num;
      printf("the number:%d",*ptr);

      return 0;
  }
  在这个程序里面，本来给ptr分配了一块内存，
  但是后来又把num的地址给了ptr,那么动态分配的内存就找不着了，丢失了。

  > 在运用malloc函数时，并不会进行初始化，所以我们使用<string.h>库中的memset函数，
  > memset(指针，放入的东西，大小)
  > memset(ptr,0,sizeof(int)*N)类似的这种的。
  > 所以我们使用calloc函数。

4. calloc
  1. 函数原型：
    - void *calloc(size_t nmemb, size_t size);
    就是申请了nmemb个长度为size的连续内存空间(即总尺寸为nmemb*size),并且这些
    内存空间全部被初始化为0。

  例如 ：
    int *ptr = (int *)calloc( 8 * sizeof(int));

    int *ptr = (int *)malloc(sizeof(int));
    memset(ptr, 0, 8 * sizeof(int));

    上面两种写法是等价的。

5. realloc
  1. 函数原型：
    - void *realloc(void *ptr, size_t size);

    - realloc函数修改ptr指向的内存空间大小为size字节。
    - 如果新分配的内存空间比原来大，则旧的内存块的数据不会发生改变；
      如果新的内存空间大小小于原来的，则可能会导致数据丢失。(慎用)
    - 该函数将移动内存空间的数据并返回新的指针。

       若ptr参数为NULL，那么调用该函数就相当于调用malloc;
       若size参数为0，并且ptr参数不为NULL，就相当于调用了free.


# 堆和栈

1. 定义如下：
  - 堆是用于存放进程运行中被动态分配的内存段，它的大小并不固定，可动态扩展或缩小。
    当进程调用malloc等函数分配内存时，新分配的内存就被添加到堆上；
    当利用free等函数释放内存时，被添加的内存从堆中被剔除。

  - 栈是函数执行的内存区域，通常和堆共享一片区域。
    栈中存放局部变量，函数参数，函数返回值。

2. 区别：
  - 申请方式：
    - 堆由人为手动申请
    - 栈由系统自动分配

  - 释放方式：
    - 堆由人为手动释放
    - 栈由系统自动释放

  - 生存周期：
    - 堆的生存周期由动态申请到人主动释放为止，不同函数之间均可自由访问。
    - 栈的生存周期由函数调用开始到函数返回时结束，函数之间的局部变量不能互相访问。

  - 发展方向：
    - 堆是从低地址到高地址，和其他区段一样。
    - 栈是从高地址到低地址。

    int *ptr1 = NULL;
    int *ptr2 = NULL;

    //这两个都是局部变量，所以本身的地址(&ptr)在栈中。

    ptr1 = (int *)malloc(sizeof(int));
    ptr2 = (int *)malloc(sizeof(int));

    //动态分配完后，所指向的地址(ptr)在堆中。

    printf("Stack: %p->%p\n",&ptr1,&ptr2);
    printf("heap: %p->%p\n",ptr1,ptr2);

    输出：
    Stack: 00000000005ffe88->00000000005ffe80
    heap: 0000000000713bc0->0000000000713c00

    可见，栈是从高到低，堆是从低到高。(栈更环保)

# 结构体

  若改为这样，就变成了占用8个字节，因为前两个char共同占用了四个。
  |ch|ch|  |  |         |  |  |  |  |

5. 动态申请结构体

下面对于申请新结点时的情形进行详细的解释：
  newnode = (Node*)malloc(sizeof(Node));
    - malloc函数在堆上面申请了一块Node类型结点大小的内存，也就是sizeof(Node)
    - 然后又返回这块内存的地址，并存入newnode里面,也就是说newnode本身不是结点，只是一个指向结点内存地址的指针变量。(它也不在这个指针里面.)

还有一个"指向"与"赋值"的问题：

在平时会如下定义指针变量：
  int a = 10;
  int *p = &a;

在这里，我们会说p指针指向整型变量a.
但是，仔细观察会发现，p中存放的地址其实就是a的地址，但是我们还是用了指向这个词。
说明p指向的是一个数据元素，一个数据类型，并不是地址，
反而  p中所存放的地址  和  p所指向的数据的地址  是一样的地址。

下面就此问题在链表头插入函数的应用进行详解：
先申请了一个新结点：Node* newnode = (Node*)malloc(sizeof(Node));
然后进行具体指针操作。
头指针(L)    头结点        首元结点      第2结点
   ↓         ↓            ↓            ↓
  [L]  →  [head|next] → [data|next] → [data|next] → NULL
           (不存数据)    (存数据)         (存数据)
                                            ↑
               newnode        新结点         ↑
                  ↓            ↓            ↑
              [newnode]   →   [e|next]      ↑(指向结点)
                                  ↓         ↑
                                  ↓         ↑
                                  → → → → → → 


要让头结点指向新结点，其实就是让头结点里面的指针域存放的内存和newnode指针一样就可以了
所以进行第一步赋值，进而让头结点指向新结点。
L->next = newnode;
然后我们其实会发现，在上面的赋值过程中，L->next本身会被覆盖，这样首元结点就找不到了。
所以先放到一个临时指针变量temp里面。
Node* temp;
temp = L->next;
但是这一步要在上一步之前才行的。
然后再让  新结点  指向  之前的首元结点  就可以了，
也就是让新结点里面指针域的指针  和  原来指向之前的首元结点的指针  一样就可以了。
所以有了
newnode->next = temp;

所以完成所有过程了。
全过程：
  temp = L->next;
  L->next = newnode;
  newnode->next = temp;

再简化一下，可以先把新结点和后面的接上，再接前面的。
刚才主要是会覆盖前一个的next指针，导致找不到下一个结点的地址，所以我们先完成后面的操作。
先进行，newnode->next = L->next.
再进行，L->next = newnode.

发现，还是一样的。

# C语言的内存管理  内存池(memory pool)

-大量的使用malloc和free函数会产生大量内存碎片,所以用内存池来解决。

1. 内存池
  在一块内存空间要被free掉时，我们不适用free函数，而是把这块内存放到内存池里面，然后
  在下一次需要一块空间时，不再使用malloc函数，而是在内存池里面查找有没有内存空间，有的话就不用再
  重新分配内存了，这就减少了内存碎片的产生。

2. 使用单链表来维护一个简单的内存池
  - 只需要将没有用的内存空间地址依次用一个单链表记录下来；当再次需要的时候，从这个单链表中获取即可。
